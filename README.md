# Machine Learning Visualization Platform
## Support Vector Machine (SVM) vs K-Nearest Neighbors (KNN)

---

# Project Overview

This project is an interactive machine learning system designed to explore and compare the behavior of two classical supervised learning algorithms: **Support Vector Machines (SVM)** and **K-Nearest Neighbors (KNN)**. The system combines machine learning theory, data visualization, and a modern web interface to provide an educational platform that allows users to understand how different algorithms classify the same dataset.

Unlike traditional machine learning demonstrations that only show final accuracy values, this project focuses on **interpretability and visualization**. The models are trained on a real dataset and their behavior is illustrated through decision boundary visualizations, scatter plots, and mathematical explanations.

The system also provides a frontend interface where users can explore the dataset, read the mathematical formulation of the algorithms, and compare the predictions generated by each model.

The project is designed to help students and developers understand how machine learning algorithms behave internally and how their mathematical foundations influence their classification decisions.

---

# Objectives

The main objectives of this project are:

• Demonstrate how different classification algorithms behave on the same dataset  
• Visualize decision boundaries created by different models  
• Provide mathematical explanations of machine learning algorithms  
• Allow interactive exploration of model predictions

# Dataset

This project uses the **Iris Dataset**, one of the most famous datasets in machine learning and pattern recognition.

The dataset was introduced by statistician Ronald Fisher in 1936 and is commonly used for classification problems.

The dataset contains measurements of iris flowers from three different species.

## Classes

The dataset contains three classes:

- Iris Setosa
- Iris Versicolor
- Iris Virginica

Each class represents a different species of iris flower.

## Features

Each sample in the dataset contains four numerical features:

| Feature | Description |
|------|------|
| Sepal Length | Length of the sepal (cm) |
| Sepal Width | Width of the sepal (cm) |
| Petal Length | Length of the petal (cm) |
| Petal Width | Width of the petal (cm) |

For visualization purposes, only two features are selected so that the data can be represented in a **two-dimensional feature space**.

This allows the models' decision boundaries to be plotted and analyzed visually.

---

# Data Representation

Each sample in the dataset can be represented as a vector:

x = (x₁, x₂, x₃, x₄)

Where each component represents one of the features.

In the simplified visualization case, we use:

x = (x₁, x₂)

# Machine Learning Algorithms

This project focuses on two widely used classification algorithms:

1. Support Vector Machine (SVM)
2. K-Nearest Neighbors (KNN)

These algorithms represent two fundamentally different learning approaches.

SVM is a **model-based optimization algorithm**, while KNN is a **distance-based instance learning algorithm**.

---

# Support Vector Machine (SVM)

## Concept

Support Vector Machines are supervised learning models used for classification and regression tasks.

The main idea behind SVM is to find the **optimal hyperplane** that separates data points belonging to different classes with the **maximum margin**.

The margin represents the distance between the separating hyperplane and the closest data points from each class.

These closest data points are called **support vectors**.

---

## Hyperplane Representation

The separating hyperplane is defined as:

w · x + b = 0

Where:

w = weight vector  
x = input feature vector  
b = bias term  

The decision function used for classification is:

f(x) = sign(w · x + b)

If the output is positive, the sample belongs to one class.  
If it is negative, it belongs to the other class.

---

## Optimization Objective

SVM attempts to maximize the margin between classes.

Margin = 2 / ||w||

Therefore the optimization problem becomes:

Minimize:

||w||² / 2

Subject to:

yᵢ (w · xᵢ + b) ≥ 1

Where:

yᵢ is the class label.

---

## Kernel Trick

If the data is not linearly separable, SVM can project it into higher-dimensional space using **kernel functions**.

Common kernels include:

• Linear Kernel  
• Polynomial Kernel  
• Radial Basis Function (RBF) Kernel  

This allows SVM to handle complex non-linear classification problems.

---

## Advantages of SVM

• Effective in high-dimensional spaces  
• Works well with clear class separation  
• Robust against overfitting when properly regularized  
• Can handle non-linear boundaries using kernels  

---

# K-Nearest Neighbors (KNN)

## Concept

KNN is a **non-parametric and instance-based learning algorithm**.

Unlike many machine learning algorithms, KNN does not build a model during training. Instead, it stores the entire dataset and makes predictions based on the similarity between data points.

When a new data point needs to be classified, the algorithm looks at the **K closest samples** in the training dataset.

The class is then determined by **majority voting** among those neighbors.

---

## Distance Measurement

To determine similarity between samples, KNN calculates the distance between feature vectors.

The most common metric used is **Euclidean Distance**.

Euclidean distance between two points x and y is:

d(x,y) = √ Σ (xᵢ − yᵢ)²

Where:

xᵢ and yᵢ are the feature values.

---

## Prediction Process

Step 1  
Compute the distance between the new point and all training points.

Step 2  
Sort distances from smallest to largest.

Step 3  
Select the K nearest neighbors.

Step 4  
Assign the class that appears most frequently among those neighbors.

---

## Choosing K

The value of K is an important hyperparameter.

Small K:
- Sensitive to noise
- High variance

Large K:
- More stable
- May smooth decision boundaries too much

---

## Advantages of KNN

• Very simple to understand and implement  
• No training phase required  
• Naturally handles multi-class problems  
• Flexible decision boundaries
